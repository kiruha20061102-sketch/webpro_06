# 管理者向け仕様書


## インストールおよび起動手順

### リポジトリのクローン
ターミナルを開き，下記の操作を行いGitHubからソースコードを取得する．

\$ git clone https://github.com/kiruha20061102-sketch/webpro_06
cd webpro_06

### 依存パッケージのインストール
Node.jsがインストールされている環境で，必要なモジュール（Express，EJS）をインストールするため下記の操作をターミナルで行う．

\$ npm install

### サーバーの起動
下記の操作をターミナルで行い，各システムのサーバーを起動し，8080番ポートを立ち上げる．

\$ node onepiece.js  # または soccer.js，movie.js

### 終了方法
起動中のターミナルでCtrl + Cを押下することで，サーバープロセスを終了する．

## 起動できない場合の対処法
「EADDRINUSE: address already in use」というエラーが出る場合，指定したポートが他のアプリで既に使用されている．別のポート番号に変更して再度実行すること．起動時にエラーが出る場合は，再度 npm install を実行してパッケージの状態を確認する．

## 分かっている不具合・制限事項
本システムはデータベースを使用せず，サーバー上の配列変数にデータを保持している．そのため，サーバーを停止・再起動すると追加や編集，削除したデータはすべて初期状態にリセットされる．また，登録処理後にブラウザの戻るボタンや更新ボタンを使用すると同じデータが二重に登録されたり，他のデータが削除されたりする．
